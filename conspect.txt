0) Зачем писать "React" в stateful components? Затем, что JSX приводится в React.Component {}
1) Обязательно в циклах задаём "key" (как правило задаём по ID);
2) пишем static defaultProps - это проверка для приходящих данных в компонент (теперь данные будут всегда в ожидаемом виде,например comments : []);
3) пишем propTypes
    static propTypes = {
       article: PropTypes.shape({
          id: PropTypes.string.isRequired,
          title: PropTypes.string.isRequired,
          text: PropTypes.string.isRequired
       }).isRequired       <-- обязательные комментарии. Лмбо, можно убрать и оставить внутренние "isRequired", в таком случае сам коммент не обязателен но если прийдёт - то у него будут обязательные поля
    }
4) "react-hmre" - в .babelrc добавить в "presets" для hot loder-а (перевызывает метод "render" в компоненте)
5) "refs" - только class Components
6) почитать про коррирование!!!
